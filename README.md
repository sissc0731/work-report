## 🎯 解决什么问题

| 你的输入 | 它输出 |
|----------|--------|
| "改了3个bug" | "定位并修复了3处线上问题，涉及核心交易链路，保障了系统稳定性" |
| "开了两个会" | "参与跨部门需求对齐和技术方案评审，推动项目关键决策落地" |
| "写了点文档" | "输出XX模块技术文档，沉淀团队知识资产，降低新人上手成本" |

## 🚀 快速开始

```bash
git clone https://github.com/sissc0731/work-report.git
cp -r work-report/.claude ./
```

然后对 Claude 说：

> "帮我写今天的日报：改了登录页的bug，跟产品对了需求，写了接口文档"

---

## 📖 支持格式

| 格式 | 触发词 |
|------|--------|
| 📅 日报 | "写日报"、"今天日报" |
| 📊 周报 | "写周报"、"这周干了" |
| 📈 月报 | "写月报"、"本月总结" |
| 🎯 年终总结 | "年终总结"、"年度汇报" |

---

## 💡 示例

### 输入
```
今天做了：
- 修了登录页面验证码不显示的bug
- 跟产品经理讨论了新功能需求
- 给新同事讲了一下项目结构
- 更新了3个接口的文档
```

### 输出
```
📅 2026年6月16日 工作日报

一、核心工作

1. 【问题修复】定位并解决登录页验证码渲染异常问题
   - 排查根因：第三方组件版本兼容性问题导致Canvas初始化失败
   - 修复方案：升级依赖版本并增加降级兜底逻辑
   - 影响范围：全量用户登录场景，修复后验证码加载成功率恢复至99.9%

2. 【需求对接】参与V2.8版本新功能需求评审
   - 与产品侧对齐批量导出功能的技术可行性
   - 明确分页加载+异步导出的实现方案，避免接口超时
   - 输出技术评估时间线，预计下周一进入开发

3. 【团队建设】协助新同事完成项目环境搭建
   - 讲解项目模块划分和技术栈选型
   - 梳理新人上手指南中的缺失步骤，补充至Wiki

4. 【文档沉淀】更新用户/订单/支付模块接口文档
   - 补充了v2.7新增的参数说明和错误码
   - 同步更新Postman Collection

二、明日计划
- 开始批量导出功能的后端接口开发
- Code Review 张同学的订单模块PR

三、风险与阻塞
- 无
```
